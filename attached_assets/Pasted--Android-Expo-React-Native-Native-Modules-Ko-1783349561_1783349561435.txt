أريد بناء تطبيق Android احترافي باستخدام Expo + React Native مع إمكانية إضافة Native Modules بلغة Kotlin عند الحاجة.

التطبيق ليس مؤقتًا عادياً، بل مساعدًا لحماية السمع ومراقبة استخدام سماعات الأذن.

## الفكرة

التطبيق يجب أن يعمل تلقائيًا دون أن يضغط المستخدم أي زر.

يبدأ العد عندما يبدأ المستخدم فعلاً في الاستماع إلى الصوت عبر السماعات، ويتوقف عند التوقف عن الاستماع.

بعد مرور مدة محددة (افتراضيًا 60 دقيقة) يرسل إشعارًا يطلب من المستخدم أخذ استراحة.

---

## الهيكل المطلوب

يجب أن تكون بنية المشروع بالشكل التالي:

- Expo + React Native
- TypeScript
- Native Module بلغة Kotlin
- إمكانية البناء بواسطة EAS Build

يجب أن يكون:

React Native مسؤولاً عن:

- جميع الواجهات
- الإعدادات
- الإحصائيات
- التخزين
- التنبيهات
- الثيم
- الترجمة

أما Kotlin فيكون مسؤولاً فقط عن التعامل مع نظام Android.

---

## مسؤوليات Kotlin

أريد أن يكون Kotlin عبارة عن Event Engine.

أي أنه يستمع لأحداث النظام فقط ويرسلها إلى React Native.

مثل:

- Bluetooth headset connected
- Bluetooth headset disconnected
- Wired headset connected
- Wired headset disconnected
- Audio playback started
- Audio playback stopped

ولا يقوم Kotlin بأي منطق خاص بالواجهة.

بل يرسل الأحداث فقط إلى React Native.

---

## مسؤوليات React Native

عند استقبال الأحداث القادمة من Kotlin يقوم React Native بـ:

- بدء المؤقت
- إيقاف المؤقت
- استكمال المؤقت
- إرسال الإشعارات
- تحديث الواجهة
- حفظ الإحصائيات

---

## منطق التطبيق

الهدف هو احتساب وقت الاستماع الحقيقي.

وليس مجرد وقت اتصال السماعة.

لذلك يجب أن يكون المنطق:

إذا:

- توجد سماعة متصلة
- ويوجد تشغيل صوت

يبدأ العد.

إذا توقف تشغيل الصوت:

يتوقف العد.

إذا عاد تشغيل الصوت:

يكمل العد من آخر قيمة.

عند الوصول إلى 60 دقيقة:

يرسل إشعارًا مثل:

"لقد استخدمت السماعات لمدة ساعة، ننصح بأخذ استراحة لمدة 10 دقائق."

---

## أنواع السماعات المدعومة

أريد دعم:

- Bluetooth Headsets
- Wired Headsets

ولا أريد الاعتماد على شركة معينة.

أي سماعة تعمل كمخرج صوت يجب أن يدعمها التطبيق.

---

## الإحصائيات

أريد إمكانية عرض:

- مدة الاستماع اليوم
- مدة الاستماع هذا الأسبوع
- مدة الاستماع هذا الشهر

مع رسوم بيانية لاحقًا.

---

## الإعدادات

أريد أن يكون المستخدم قادرًا على تغيير:

- مدة التنبيه (30 / 45 / 60 / 90 دقيقة)
- مدة الاستراحة
- تفعيل أو تعطيل الإشعارات

---

## فلسفة التطبيق

أريد أن يكون Kotlin صغيرًا جدًا.

أي لا يحتوي على أي منطق خاص بالتطبيق.

بل يكون مجرد جسر بين Android و React Native.

أما جميع منطق التطبيق فيكون داخل React Native.

---

## المطلوب الآن

في هذه المرحلة لا أريد كتابة الكود مباشرة.

أريد أولاً:

1. اقتراح أفضل Architecture للمشروع.
2. اقتراح هيكل المجلدات.
3. تحديد أفضل طريقة لدمج Kotlin داخل Expo.
4. تحديد إن كانت هناك أي قيود أو Permissions يحتاجها التطبيق.
5. اقتراح أفضل أسلوب لبناء Native Module بحيث يكون قابلاً للتوسعة مستقبلاً.

بعد الاتفاق على ذلك سأطلب البدء في تنفيذ المشروع خطوة بخطوة.
